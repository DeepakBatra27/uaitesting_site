About us

A unified platform for university life.

We build software that brings disconnected campus tools under one roof — designed for IIT Delhi students, with around 7,000 students in scope at the platform's current stage.

uaitesting develops a campus-wide AI platform that unifies the everyday services a university student needs into a single interface — backed by an in-house large language model service, vector search, and real integrations with institute systems like LDAP, IMAP email, library OPAC, and timetable data.

At its core sits Horizon, our conversational AI agent, which understands intent across structured data (timetables, courses, emails) and unstructured data (library catalog, hospital information). Around Horizon, we layer focused modules — each previously a standalone tool — so students no longer juggle five different portals to get through a single day.

We treat the campus as a real production environment: LDAP-only authentication means we never store passwords, JWT-secured microservices isolate failure domains, and every AI interaction routes through a centralized inference service so we can reason about cost, latency, and safety as one system rather than ten.

What we do

Six AI-powered modules. One sign-in.

Every module shares the same authentication, the same design language, and the same intelligence layer — so behavior and quality stay consistent across the platform.

Chat · Horizon

Conversational AI agent

Horizon answers campus questions across courses, timetables, emails, library catalog, and hospital info — with hybrid vector + keyword search and conversation memory.

Email

Smart inbox

Pulls real IITD IMAP email, classifies urgency, surfaces deadlines, and lets Horizon answer "when is the FEE deadline?" by reading actual messages — never by guessing.

Timetable

Real schedules

Generates a student's actual weekly timetable from institute course data, with semester week numbering, conflict detection, and personal edits.

Library

Searchable OPAC

A resilient frontend on top of the institute library — fuzzy and spell-corrected search, response caching, and a circuit breaker for the upstream catalog.

Lost & Found

Image-based matching

Visual search powered by DINOv2 image embeddings so a photo of a lost item can find its match — text descriptions optional, not required.

Hospital

Information bot

A retrieval-augmented assistant for institute hospital information — clinic hours, services, emergency contacts — grounded in source documents, not model memory.

What we plan to do

Where uaitesting is headed.

We're early. The platform works end-to-end today; the next phase is depth, reach, and trust.

  • 01

    Deepen Horizon's understanding

    Improve cross-module reasoning so a single question can pull from courses, emails, and timetable in one answer — with pronoun resolution and persistent memory.

  • 02

    Expand beyond IIT Delhi

    Generalize the institute integrations (LDAP, IMAP, OPAC, hospital RAG) into pluggable connectors so other universities can adopt the platform without rebuilding it.

  • 03

    Self-hosted LLM service

    Move from a managed inference provider to a self-hosted model where it makes sense — for cost, latency, and data residency — while keeping the centralized inference contract.

  • 04

    Safety, observability, and assurance

    Build out evaluation harnesses, drift detection, and audit-grade logging so the platform stays trustworthy as it scales across campuses.
